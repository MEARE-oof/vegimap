
The api helps handle requests.
Think of them as function that you call from the client to the server.
The server then does something and returns a response to the client.

In order to use the api, you need to know the endpoint of the api.

also, here's a template for requests:
fetch("/api/endpoint", {
    method: "POST",
    headers: {
        "Content-Type": "application/json"
    },
    body: JSON.stringify({
        //data
    })
}).then(res => {
    if (res.ok) {
        return res.json();
    } else {
        throw new Error();
    }
}).then(data => {
    // do something with the data
}).catch(err => {
    // do something with the error
});

Documentation:
    /api/register - registers a user, and returns the id of the user
        args:
            username - the username of the user
            password - the password of the user
            email - the email of the user
        returns:
            message - the message of the response
            token - the token of the user

    /api/login - logs in a user, and returns the id of the user
        args:
            email - the username of the user
            password - the password of the user
        returns:
            message - the message of the response
            token - the token of the user

    /api/markers/add - adds a marker to the database
        args:
            token - the token of the user
            lat - the latitude of the marker
            lng - the longitude of the marker
            title - the title of the marker
            description - the description of the marker
        returns:
            message - the message of the response
            id - the id of the marker

    /api/markers/list - get a list of markers
        args:
            token - the token of the user
        returns:
            message - the message of the response
            markers - the list of markers
